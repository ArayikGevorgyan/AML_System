# Test Plan — AML Transaction Monitoring System

## 1. Scope
Testing covers all backend API endpoints, the AML rules engine, sanctions screening, and core frontend workflows.

## 2. Test Types

| Type | Description | Tools |
|------|-------------|-------|
| Unit Testing | Individual service functions | pytest |
| Integration Testing | API endpoints with real DB | pytest + httpx |
| Manual Testing | UI workflows via browser | Chrome DevTools |
| Performance Testing | Rules engine and sanctions search speed | Python time module |

## 3. Test Environment
- Backend: Python 3.13, SQLite with seeded test data
- Frontend: React dev server via npm start
- Test data: Generated by seed_data.py script

## 4. Entry and Exit Criteria

| Criteria | Description |
|----------|-------------|
| Entry | Backend running, seed data loaded, frontend accessible |
| Exit | All P1 test cases pass; no critical bugs open |

## 5. Test Schedule

| Phase | Activity |
|-------|---------|
| Week 6, Day 1-3 | Write and run all test cases |
| Week 6, Day 4-5 | Log bugs, fix, retest |
| Week 7, Day 1 | Write final test summary report |

## 6. Roles and Responsibilities

| Role | Responsibility |
|------|---------------|
| Developer | Write test cases, execute tests, fix bugs |
| Supervisor | Review test results and approve final report |

## 7. Risk Areas for Testing
- Rules engine accuracy: verify each of the 8 rules triggers correctly
- Sanctions screening: verify fuzzy matching returns correct results
- Role-based access: verify 403 errors for unauthorized actions
- Email verification: verify code delivery and validation
